# POS 오픈 API 표준안 제안 (Proposed Open POS API Standard)

본 시스템이 수많은 상점에서 범용적으로 사용되기 위해 POS 제조사 및 소프트웨어 벤더에 제안하는 표준 인터페이스 규격입니다.

## 1. 개요
POS 업체가 아래의 API 규격만 열어주면, 중계 플랫폼은 별도의 하드웨어 연동 없이도 주문을 POS로 직접 전달할 수 있습니다.

## 2. 핵심 API 엔드포인트 (Proposed)

### [POST] `/api/v1/orders/external`
- **목적**: 외부(QR 오더)에서 발생한 주문을 POS 시스템에 등록
- **Request Payload**:
  ```json
  {
    "order_id": "QR-20260311-001",
    "table_no": "T-05",
    "items": [
      {"menu_id": "M001", "name": "김치찌개", "price": 9000, "qty": 1},
      {"menu_id": "M005", "name": "공기밥", "price": 1000, "qty": 2}
    ],
    "total_amount": 11000,
    "payment_type": "PREPAID_VAN",
    "van_transaction_id": "VAN-99887766"
  }
  ```

### [GET] `/api/v1/van/status`
- **목적**: 현재 POS에 연결된 VAN 단말기의 상태 및 승인 가능 여부 확인

## 3. POS 업체의 기대 실익
1.  **플랫폼 경쟁력 강화**: 자체 모바일 오더를 개발하지 않고도 수많은 QR 오더 서비스와 연동되는 '열린 POS'라는 브랜드 이미지 구축.
2.  **데이터 가치 증대**: 외부 주문 데이터가 POS로 통합되어, 점주에게 더 정확하고 풍부한 매출 분석 리포트 제공 가능.
3.  **H/W 부담 경감**: 무전력/무기기 주문 방식인 QR 오더를 수용함으로써, 기존 태블릿 오더의 AS 관리 비용 절감.

## 4. 결론
POS 업체가 API라는 **'작은 통로'**만 열어준다면, 식당은 **'월 3만원'**이라는 파격적인 고정비로 디지털 전환이 가능해지며 이는 외식업 생태계 전체의 상생으로 이어질 것입니다.

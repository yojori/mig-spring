# 개인 식당 POS 환경 분석 (Individual POS Environment Analysis)

프랜차이즈가 아닌 일반 개인 식당(소상공인)의 POS 환경을 분석하여, 사용자님의 '통합 세트' 모델이 침투할 수 있는 기술적/사업적 지점을 도출합니다.

## 1. 하드웨어 및 OS 환경
- **기기**: 고사양보다는 내구성이 강조된 일체형 POS 단말기가 주류. (Windows 7/10 기반이 대다수)
- **주변기기**: 
    - **영수증 프린터**: Serial(RS-232) 또는 USB로 연결된 열전사 프린터.
    - **VAN 단말기**: POS와 연동되어 카드를 긁거나 꽂는 외장형 단말기.
    - **돈통 (Cash Drawer)**: 프린터 출력 신호(Kick-out)에 따라 열리는 구조.

## 2. 소프트웨어 및 데이터 구조
- **폐쇄적 DB**: 로컬 DB(MSSQL, SQLite 등)를 사용하며, 외부에서 데이터를 가져가거나 넣기 매우 어렵게 설계됨.
- **업데이트 지연**: 인터넷 연결은 되어 있으나 보안 프로그램이나 오래된 윈도우 환경 때문에 웹 기술 기반의 최신 앱 연동이 까다로움.
- **벤더 가두리**: POS 프로그램 제작사(예: 오케이포스)가 카드 결제사(VAN)와 계약되어 있어, 다른 결제 수단을 붙이려면 높은 기술료나 수수료를 요구함.

## 3. 사용자 아이디어(통합 세트)의 적용 포인트
개인 식당 점주가 느끼는 가장 큰 장벽을 다음과 같이 해결합니다.

| 페인 포인트 (Pain Point) | 통합 세트의 해결책 (Integrated Solution) |
| :--- | :--- |
| **"POS 따로, 주문앱 따로 너무 비싸요"** | **단일 패키지**: POS 프로그램 안에 모바일 주문 수신 기능이 기본 내장됨. |
| **"메뉴 수정이 너무 번거로워요"** | **실시간 동기화**: POS에서 메뉴 가격 바꾸면 QR 주문 웹 메뉴판도 즉시 반영. |
| **"정산이 안 맞아요"** | **데이터 단일화**: POS 매출과 폰 주문 매출이 같은 DB에 기록되어 오차 제로. |
| **"관리가 힘들어요"** | **단일 창구**: AS나 장애 처리를 한 곳에서 통합 지원. |

## 4. 결론: "개인 식당을 위한 클라우드 기반 경량 POS"
개별 식당 환경은 하드웨어는 낙후되어 있고 소프트웨어는 폐쇄적입니다. 사용자님의 의도대로 **'POS SW + 모바일 주문 웹'을 한 세트**로 묶어, 기존 무거운 윈도우 POS를 대체할 수 있는 **'웹/클라우드 기반 경량 POS 패키지'**로 접근한다면 소상공인 시장을 빠르게 장악할 수 있을 것입니다.

# 3번 옵션: 아주 쉬운 시나리오 설명

사용자님이 식당을 차렸을 때를 가정한 **"협력 없는 독자 연동"**의 실제 모습입니다.

### 1. 상황 (Scenario)
- 사용자님이 식당을 오픈하고, 시중에서 흔히 파는 **일반적인 POS(예: 오케이포스)**를 한 대 샀습니다. 
- 하지만 그 POS 업체와는 아무런 기술 협의를 하지 않았습니다.
- 사용자님이 직접 만든 **'핸드폰 주문 소프트웨어'**만 가지고 연동을 하고 싶습니다.

### 2. 해결 방법: "비서 프로그램(Agent) 심기"
흔히 쓰는 POS는 사실상 **'윈도우가 깔린 컴퓨터'**입니다. 사용자님이 만든 소프트웨어를 연동하는 방법은 다음과 같습니다.

1.  **에이전트 설치**: 사용자님이 만든 아주 작은 '비서 프로그램(에이전트)'을 그 POS 컴퓨터에 깔아둡니다.
2.  **주문 전달**: 고객이 폰으로 주문하면, 클라우드를 통해 이 '비서 프로그램'으로 신호가 압니다.
3.  **포스에 밀어넣기 (핵심)**: 신호를 받은 비서 프로그램이 POS 화면에 주문을 대신 입력해 줍니다.
    - **자동 입력 방식**: 사람이 키보드로 치는 것처럼 비서 프로그램이 POS의 메뉴 버튼을 초고속으로 대신 '클릭'해 줍니다. 
    - **DB 저장 방식**: POS 프로그램이 사용하는 데이터베이스 파일에 주문 내역을 몰래 '슥' 집어넣습니다.

### 3. 결제는 어떻게?
- 고객이 폰에서 삼성페이 등으로 결제하면, 비서 프로그램이 POS와 연결된 **'카드 단말기'**에 "결제 완료되었으니 영수증 뽑아라!"라고 신호를 직접 보냅니다.
- 결과적으로 점주는 POS 업체와 아무 대화를 안 했음에도 불구하고, 고객의 폰 주문이 자기 POS에 척척 찍히는 것을 보게 됩니다.

### 4. 한 줄 요약
**"POS 프로그램의 눈(UI)이나 뇌(DB)를 직접 건드리는 '비서 프로그램'만 하나 잘 만들면, POS 업체 허락 없이도 무엇이든 연동할 수 있습니다."**

이것이 바로 배달의민족이나 요기요가 쓰고 있는 **'주문 접수 프로그램'**의 원리이며, 사용자님도 충분히 하실 수 있는 방식입니다.

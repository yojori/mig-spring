# 기술 아키텍처 및 비용 최소화 전략 (Architecture & Cost Strategy)

본 시스템은 상점이 추가 하드웨어 없이 기존의 인터넷 환경만으로 운영될 수 있도록 설계되었습니다.

## 1. 하드웨어 구성: "Zero Table Device + Integrated POS"
- **테이블**: QR 코드 스티커 (고객 BYOD 활용).
- **매장**: 당사가 직접 제공하거나 우리 소프트웨어가 최적화된 **통합 POS 유닛**.

## 2. 시스템 논리 구조 (Integrated Ecosystem)
1.  **Unified Backend**: POS 관리 데이터와 모바일 주문 데이터가 분리되지 않고 하나의 통합 클라우드에서 관리됩니다.
2.  **Native Linkage**: 
    - 고객 주문 발생 시, 외부 중계 프로그램 없이 POS 소프트웨어 엔진이 실시간으로 신호를 직접 수신합니다.
    - 결제 요청 시, 모바일 앱이 POS 내부의 결제 라이브러리를 직접 호출하여 상점의 VAN 단말기로 결제를 승인합니다.
3.  **Revenue Struct**: 결제 자체에 대한 정액/요율 수수료(POS 측) + 모바일 오더 서비스 이용료(월 3만원 수준)를 하나의 통합 패키지로 부과합니다.

## 3. 수익 모델 및 비용 전략
- **No Commission**: 결제 금액에 따른 수수료를 1%도 가져가지 않습니다.
- **Fixed Fee**: 인프라 유지 및 중계 서비스 이용료로 **월 30,000원**의 고정 비용만 청구합니다.
- **TCO (총소유비용) 절감**: 하드웨어 리스료 + 유지보수비 + 수수료를 모두 합쳤을 때 기존 모델 대비 70% 이상 저렴한 구조를 추구합니다.

## 4. 유니버설 연동 기술 (Cooperation-Free Integration)
POS 업체의 공식 협조가 없어도 연동을 가능케 하는 기술적 핵심입니다.

- **DB Mirroring/Infection**: POS의 로컬 DB 구조를 분석하여 주문 데이터를 직접 삽입(Injection)하는 기술.
- **Virtual Printer Driver**: POS가 주문서를 출력할 때 가상 프린터 드라이버가 신호를 가로채서 모바일 오더 데이터와 매칭/처리하는 방식.
- **Window Message Hooking**: 윈도우 기반 POS 프로그램의 특정 이벤트(주문창 호출 등)를 감지하여 자동으로 입력값을 채워넣는 자동화 기술.

## 5. 단계별 기술 확장성
- **Lv.1 (Manual Bridge)**: 주방 프린터만 공유하여 주문서만 출력 (가장 낮은 단계)
- **Lv.2 (Semi-Auto Agent)**: 윈도우 에이전트를 통한 주문 자동 트리거 및 VAN 단말기 중계.
- **Lv.3 (Deep Native)**: POS 업체와 공식 제휴하여 소스 코드 레벨에서 모듈로 탑재 (가장 높은 단계)
